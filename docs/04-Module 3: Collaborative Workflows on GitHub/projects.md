# Understanding GitHub Projects

GitHub contains a massive range of open-source projects. GitHub additionally provides an entire suite tools for version control, issue tracking, and code review. A key feature of GitHub is GitHub projects which serves as a one-stop-shop for a codebase.

## What is a GitHub Project?

A GitHub Project is a collaboration tool within GitHub repositories, designed to help organize and prioritize your work. You can use GitHub Projects to manage workflows, track issues, pull requests, and notes within a repository or across repositories.

## Key Components of a GitHub Repository

Here’s an overview of the different parts of a GitHub repository:

### Projects

Projects on GitHub allow you to organize your work with boards customizable to various workflows. On these boards, you can add issues and pull requests, track timeline, and assign participants.
  ![GitHub Projects](/img/projects.png)

## How to Create a GitHub Project

Here is how you can create a GitHub project.

1. Go to the main page of the repository.

2. Click on the 'Projects' tab at the top.

3. Click on 'New project'. Select a name and choose a template.

4. Add columns for tasks, or create a roadmap for a timeline.

5. Add issues, pull requests, or notes to your project.
